# v0.2: JavaScript => TypeScript

## 迁移说明

将 `ai4js/nn` 下的所有 `.js` 文件转换为 `.ts`，加入完整的 TypeScript 类型标注。

## 变更内容

### 文件转换

| .js | .ts |
|-----|-----|
| tensor.js | tensor.ts |
| nn.js | nn.ts |
| gpt.js | gpt.ts |
| chargpt.js | chargpt.ts |
| index.js | index.ts |
| chargpt_main.js | chargpt_main.ts |

### 新增配置文件

- `tsconfig.json` - TypeScript 配置
- `package.json` 更新 - 添加 TypeScript 相关依赖与 Jest ts-jest 配置

### TypeScript 配置

```json
{
  "compilerOptions": {
    "target": "ES2020",
    "module": "commonjs",
    "strict": true,
    "esModuleInterop": true,
    "skipLibCheck": true,
    "sourceMap": true
  },
  "include": ["src/**/*"],
  "exclude": ["node_modules", "dist", "tests"]
}
```

### Jest ts-jest 配置

```json
{
  "preset": "ts-jest",
  "testEnvironment": "node",
  "transform": {
    "^.+\\.tsx?$": ["ts-jest", { "diagnostics": false }]
  },
  "moduleNameMapper": {
    "^(\\.{1,2}/.*)\\.js$": "$1"
  }
}
```

### 类型处理策略

为避免 ndarray 的泛型类型与 strict mode 冲突，采用宽鬆类型处理：

```typescript
// 避免泛型问题，直接使用 ndarray 返回的 NdArray 类型
data: NdArray;
grad: NdArray;

// 避免 Parameters<typeof Tensor> 的复杂推断
const otherT = other instanceof Tensor ? other : new Tensor(other as Float32Array | number | unknown[]);
```

### GPT.__call__ 返回类型

显式标注 tuple 返回类型（修复之前的错误推断）：

```typescript
__call__(idx: Tensor, kv_caches: [Tensor, [Tensor, Tensor]][] | null = null) {
  // ...
  return [logits, new_caches];
}
```
